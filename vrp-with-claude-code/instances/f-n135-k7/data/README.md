- https://vrp.galgos.inf.puc-rio.br/index.php/en/

## データ構造

### インスタンス: F-n135-k7

| 項目 | 値 | 説明 |
|------|-----|------|
| 問題タイプ | CVRP | 容量制約付き配送計画問題 |
| ノード数 | 135 | デポ1個 + 顧客134個 |
| 車両台数 | 7 | 使用可能な車両数 |
| 車両容量 | 2210 | 各車両の最大積載量 |
| 距離計算方式 | EUC_2D | 2次元ユークリッド距離 |
| 最適値 | 1162 | 既知の最適解のコスト |

## ファイル形式

### VRPファイル (.vrp)

VRPファイルは以下のセクションで構成される

#### ヘッダー情報

- `NAME`: F-n135-k7
- `COMMENT`: Fisher: problem 12, No of trucks: 7, Optimal value: 1162
- `TYPE`: CVRP（Capacitated VRP）
- `DIMENSION`: 135
- `EDGE_WEIGHT_TYPE`: EUC_2D
- `CAPACITY`: 2210

#### NODE_COORD_SECTION

ノードの座標情報（ノード番号、X座標、Y座標）
- 135個のノード座標が定義される
- ノード1がデポ（開始・終了地点）

#### DEMAND_SECTION

各ノードの需要量
- ノード1（デポ）の需要量: 0
- 顧客ノードの需要量: 1〜550の範囲

### 解ファイル (.sol)

最適解のルート情報
- 7つのルート
- 総コスト: 1162

## データ統計

### 規模

- **問題規模**: 中規模（135ノード）
- **f-n45-k4との比較**:
  - ノード数: 3倍（45→135）
  - 車両数: 1.75倍（4→7）
  - 最適コスト: 1.6倍（724→1162）

### 座標範囲

- X座標: 約 -50 〜 50
- Y座標: 約 -50 〜 50

## 制約条件

1. **容量制約**: 各ルートの総需要量 ≤ 車両容量（2210）
2. **訪問制約**: 各顧客は必ず1回だけ訪問
3. **デポ制約**: 全ルートはデポから開始し、デポに戻る
4. **車両数制約**: 使用車両数 ≤ 7台

## 計算上の注意点

- **距離計算**: TSPLIBの標準に従いnint()またはDecimal.quantize(ROUND_HALF_UP)を使用
- **インデックス**: .solファイルのノード番号は.vrpファイルより1小さい（.sol「1」= .vrp「2」）
